E-commerce App with Improved Accesibility

Project Overview

After the 4-week UX/UI & Accessibility Design course organized by Code First Girls, students had a chance to face a challenge hosted by Brandwatch. There were two objectives of the challenge:

  • to perform an accessibility audit on a e-commerce website selected by a team
  • using the knowledge gained after audit: to design (and build) a new website with aspects to enhance accessibility

My contribution

  • performing a technical audit to find accessibility shortcomings
  • testing of the prototype

Audit

We decided to base our project on a well-known store's e-commerce website: Flying Tiger Copenhagen, which is a one-stop shop, where users can get everything from: homeware, electronics, school and office supplies, toys and much more.
These are issues we identified:
Over-Crowding
The website has an unnecessarily overcrowded header with five different icons. Example the wishlist icon (heart) becomes an overpopulated icon. The enquiry mailbox is already available in the footer and navigation hence no need for additional option to be present.
Option Overload
Due to overcrowding and too many options, it is highly likely that in user journeys “overchoice” occurs. The option overload causing the consumer to become overwhelmed thinking about the benefits and risk of their potential choices.
No Speech-To-Text Feature
The website is accessible from a language perspective but it needs to be improved to be made more accessible for neurodivergent consumers. The search bar has no speech-to-text feature which is essential for dyslexic consumers to carry out their search.
Keyboard Accessibility
There is also no keyboard accessibility for people who have tremors and difficulty controlling mousepad.
Source Code
Issues with source code that makes a website hard to be captured by screen readers.
  • List items "li" are not contained within "ul" or "ol" parent
    • elements.
  • ARIA input fields do not have accessible names
  • Links do not have a discernible name
  • Heading elements are not in a sequentially-descending order
  • Many of the images lack alt texts
